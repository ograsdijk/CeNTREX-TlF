# Work log — 2026-05-04

Two cleanup tasks on top of the existing Phase A perf work:

1. Make the fringe notebooks self-contained (parameters + computation visible
   in the notebook, not hidden behind a `.npz` load).
2. Investigate the apparent ~110 Hz mismatch between `F_RES = 119.64 kHz` and
   the visible fringe centerline in the cached far-frequency scan.

Headline result on (2): **`F_RES = 119.64 kHz` is correct**. The mismatch is a
**phase-convention artifact**: the simulator's `phi₂` parameter is the
absolute carrier phase in `cos(ωt + φ₂)`, not the rotating-frame Δφ. With the
existing setting `phi₂ = +π/2`, the rotating-frame Δφ is 0, so the molecule is
fully *transferred* at resonance instead of split 50/50. To get the
experimentally-intended 50/50 behavior, set the simulator's `phi₂ = 0` (or π).

   No other transition with magnetic-x coupling sits within ±1 kHz of F_RES.

### Why the asymmetric fringe contrast

With Δφ_RWA = 0 (the simulator's `phi₂ = π/2` setting), the survival pattern
near F_RES has period 73.6 Hz with deep minima on both sides — but contrast
falls off faster on the +δ side than on the −δ side. The +δ minima get
shallower (0.14 at +73.6 Hz; 0.35 at +147.2 Hz) while the −δ minima stay deep
(0.003 at −73.6 Hz; 0.024 at −147.2 Hz). The cause is most likely a finite-
pulse-shape AC-Stark-like effect from the rounded-rectangle envelope (each
1.6 ms pulse has a non-trivial spectral tail), but quantifying it is not
needed once the user sets `phi₂ = 0` — the pattern is then symmetric around
F_RES with a 50/50 cross at δ = 0 and ±73.6 Hz spaced sign-changing fringes.

## Recommendation for production use

Add an explicit `phi2_RWA` parameter to
[`benchmarks/run_v3b_cpu_scans.py`](benchmarks/run_v3b_cpu_scans.py) that
internally sets `phi₂_simulator = phi2_RWA + π/2`. That eliminates the
calibration step in every notebook that uses the scan helpers. Until that
lands, set the simulator's `phi₂ = 0` to get the 50/50-at-resonance Ramsey
geometry.

The deeper fix is to also expose `phi1_RWA` and to document this convention
explicitly in the `MagneticRFRegion` docstring — `cos(ωt + φ)` does NOT
correspond to "Δφ in the rotating frame" without the carrier-phase offset.

## Time spent

A couple of hours including a 2.7-hour scan that was wasted because of the
multiprocessing/monkey-patch issue: my main-process patch of `_set_rf_omega`
did not propagate to `spawn`-mode worker processes (they re-import the module
fresh on startup). The "φ₂ = 0 scan" produced by that bad run was identical to
the existing `phi₂ = π/2` cache. Lesson: with `multiprocessing.spawn`, modify
the *cfg object* in the main process before pickling for workers, or add a
parameter to the script — never monkey-patch.
